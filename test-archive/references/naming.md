# Naming Conventions Reference

## 1. 目录结构（强制）

```
wkq/                              # 根目录，固定名 wkq
├── 01_<简述>/                    # 批次目录，两位编号起
│   ├── plan/                     # 方案文档
│   ├── data/                     # 原始数据、日志、截图、接口返回
│   └── report/                   # 结论性报告
├── 02_<简述>/
│   ├── plan/
│   ├── data/
│   └── report/
└── ...
```

**子目录职责不可混**：

| 子目录 | 放什么 | 不放什么 |
|---|---|---|
| `plan/` | 测试方案文档 | 测试日志、报告 |
| `data/` | 接口返回、日志、截图、压测原始数据 | 方案、报告 |
| `report/` | 结论性报告 | 过程日志、原始返回 |

## 2. 批次目录命名

格式：`NN_<简述>` 或 `NNN_<简述>`

- **NN**：两位编号，01 起，补零。超过 99 自动转三位（100, 101…）。
- **简述**：中文短语或英文短语，词间用 `_` 连接。**不含空格、斜杠**。
- 合法示例：
  - `01_登录接口回归测试`
  - `02_并发压测`
  - `03_search_api_e2e`
  - `15_v1.2_release_smoke`
- 非法示例：
  - `1_登录`（编号要补零 → `01_登录`）
  - `01 登录测试`（含空格 → `01_登录测试`）
  - `01/login`（含斜杠）

## 3. data/ 文件命名

- 必须**以用例编号开头**：`caseNN_<描述>.<ext>`
- 仅使用小写字母、数字、下划线；扩展名可含小写字母
- 合法示例：
  - `case01_output.log`
  - `case02_response.json`
  - `case03_screenshot.png`
  - `case05_db_dump.sql`
- 非法示例：
  - `Case01_x.log`（大写 → `case01_x.log`）
  - `登录日志.log`（中文 → `login.log` 或 `case01_login.log`）
  - `结果.json`（同上）

**例外**：`deploy.log`、`env.txt` 这类跨用例的过程性文件可不用 `caseNN_` 前缀。

## 4. plan/ 与 report/ 文件命名

默认文件名：

| 子目录 | 默认文件名 |
|---|---|
| `plan/` | `test_plan.md` |
| `report/` | `test_report.md` |

若一轮方案拆成多份（如 `接口测试方案.md` + `性能测试方案.md`），允许并存，但**至少有一份 `test_plan.md` / `test_report.md`** 作为索引。

## 5. 编号生成与冲突处理

- `new` 子命令会扫描 `wkq/` 下所有 `NN_*` 目录，取最大编号 +1。
- 若发现已有三位编号（≥ 100），新建的也用三位。
- 若扫描到非法目录名（如 `tmp`、`backup`），**忽略不报错**，但不会被计入编号。
- 不要手动 mkdir 批次目录——绕过 CLI 会破坏编号自增性。

## 6. 复用批次

- 用户说"继续跑之前的方案" → 用 `resume <编号或简述>` 定位批次，把新内容**追加**到对应 `plan/` `data/` `report/`。
- 同一批次里若有多轮执行，data/ 文件用 `caseNN_runM_*` 区分，如 `case01_run2_output.log`。
- report/ 可保留历史版本：`test_report.md` 始终是最新版，旧版本另存为 `test_report_v1.md` 等。
